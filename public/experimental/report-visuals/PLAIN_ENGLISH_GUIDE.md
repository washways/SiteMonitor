# Plain-English Guide to the Visualization Layer

This experimental visualization layer is meant to make the analytics report easier to understand for operational decisions.

You do not need to upload data first. Each page can now run a fresh multi-borehole cohort analysis directly from the existing telemetry pipeline.

## What each page is for

### Network overview
Use this page to answer:
- how many sites are active right now
- which sites are telemetry-ready
- which sites are most urgent to review
- which sites have poor support or repeated downtime-like behaviour

### Borehole detail
Use this page to answer:
- what the borehole has been doing day by day
- whether pumping, groundwater level, drawdown, and specific capacity look stable or concerning
- what the current interpretation and suggested action are

### Cross-site comparison
Use this page to answer:
- which sites stand out as most stressed
- which sites are most downtime-prone
- which sites appear to be declining
- which sites currently show the best supported specific capacity

## Important caution

These visuals are **review tools**, not automatic diagnoses.

If a borehole is labelled:
- declining performance — it deserves comparison with earlier behaviour
- stressed — it deserves closer operational review
- insufficient data — do not over-interpret the charts
- unreliable or possible fault — check telemetry, power, and field condition first

The goal is to help teams decide where to look next, while keeping uncertainty visible.
