Remote Monitoring Methodology

This page explains how SiteMonitor uses flow rate, groundwater depth, drawdown, pumping duration, pumped volume, and specific capacity to turn raw telemetry into practical borehole performance intelligence. The aim is to help teams identify failing assets early, prioritise maintenance, track trends over time, and support evidence-based WASH management.

Why remote monitoring matters

Remote monitoring shifts borehole management from reactive repair to proactive performance tracking. Instead of waiting for a community complaint or a failed visit, the monitoring team can see how each site is behaving day by day and event by event.

  • Detect underperforming pumps earlier
  • See whether declining water production is linked to mechanical issues or groundwater stress
  • Track service continuity and availability across many sites at once
  • Prioritise maintenance teams based on evidence rather than guesswork
  • Build a performance history for donors, operators, and local managers
  • Assess whether interventions improve borehole efficiency over time
Good remote monitoring does not just show whether a pump is on. It helps explain how the source is responding while the pump is operating and whether that response is improving or deteriorating.

What the different readings mean

Reading What it represents How to interpret it Why it matters operationally
Flow rate Instantaneous or hourly pumping output Higher values mean more discharge. Sharp drops may indicate pump, power, blockage, or source issues. Helps confirm whether a borehole is producing enough water for demand.
Total volume Total water abstracted over a period or event High totals suggest strong use or long pumping periods. Low totals may indicate outage, low demand, or reduced capacity. Useful for service reporting and asset utilisation monitoring.
Water level above pump Groundwater level relative to the pump intake reference When this value falls during pumping, the aquifer is being drawn down. Shows how strongly the source responds to pumping.
Drawdown Difference between the start and end water level during an event Large drawdown means the borehole needs more level change to sustain pumping. Helps identify stress, inefficiency, or possible deterioration.
Maximum drawdown Worst water-level decline reached within an event Higher values indicate the strongest stress during pumping, even if the final reading recovers slightly. Important for risk screening and identifying extreme operating conditions.
Specific capacity Flow rate divided by drawdown Higher values usually reflect better borehole efficiency. Falling values can be a warning sign. Excellent for comparing boreholes and tracking performance changes over time.
Recovery How the water level rebounds after pumping Slow recovery may suggest aquifer stress or impaired performance. Supports maintenance prioritisation and longer-term source assessment.

Methodological approach used in SiteMonitor

1. Collect telemetry

The system reads remote monitoring data from both the DCP Water API and the SonSetLink API. DCP provides hourly flow and water-level series. SonSetLink provides site metadata, daily usage values such as deflow and flow1, and sensor slot arrays such as sensor2 that can be interpreted as depth patterns through the preceding day.

SonSetLink event metrics are presented as screening-level estimates. They are useful for remote monitoring, prioritisation, and trend review, but they should be interpreted as approximate because the API is daily rather than truly hourly for most sites.

2. Detect pumping events

A new event begins when flow rises above the operational threshold of 0.1 m³/h. The event continues while pumping remains above that threshold and closes once flow drops back to zero or near-zero.

3. Record start conditions

At the beginning of each event, the routine stores the first valid water-level reading. This becomes the reference condition against which the event drawdown is measured.

4. Track changes through the event

While the event is open, the algorithm accumulates volume, tracks duration, saves the latest valid flow reading, and identifies the lowest groundwater level reached during pumping.

5. Calculate performance metrics

When the event closes, SiteMonitor computes event duration, total pumped volume, end water level, drawdown, maximum drawdown, and specific capacity. These values are then presented in tables and borehole summary views.

How specific capacity is used

Specific capacity is an operational efficiency indicator. It answers a practical question: how much water is being produced for each metre of groundwater drawdown?

Specific capacity = final event flow ÷ event drawdown

Although this is not a substitute for a formal pumping test, it is extremely valuable for remote operational screening because it allows consistent comparison:

  • between one event and the next for the same borehole
  • between boreholes in the same programme
  • before and after maintenance or rehabilitation
  • between wet- and dry-season performance

How remote monitoring improves management

  1. Earlier warning: abnormal drops in flow or worsening drawdown can be spotted before full failure occurs.
  2. Better maintenance targeting: technicians can be sent first to the sites showing the strongest evidence of deterioration.
  3. Reduced travel waste: site visits can be prioritised using data rather than fixed schedules alone.
  4. Clearer reporting: project managers can show evidence of uptime, use, and performance trends.
  5. Asset learning: repeated event analysis builds a history that helps distinguish one-off anomalies from real long-term decline.
  6. Improved sustainability oversight: persistent deep drawdown or poor recovery can support decisions about abstraction pressure and source protection.

Quality control and interpretation notes

The system keeps flagged events visible so that users can see the full operational context. Missing levels, very small drawdown, irregular timestamps, or end-of-window events may reduce confidence in the final efficiency estimate.

These outputs are intended for operational monitoring, screening, and trend detection. They support decision-making but do not replace a dedicated hydrogeological test programme.
